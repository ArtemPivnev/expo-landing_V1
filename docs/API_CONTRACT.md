# Контракт API для лендинга

Ниже — ожидаемые формы данных. Сейчас фронтенд использует мок (если `apiBaseUrl` пустой), но структура подготовлена под подключение.

## 1) Получение данных мероприятия

`GET {apiBaseUrl}/events/{eventId}`

### Ответ (JSON)
Пример:
```json
{
  "companyName": "Компания-дистрибьютор",
  "brandName": "Компания",
  "contact": { "email": "sales@company.ru", "phone": "+7 (495) 000-00-00" },
  "event": {
    "eyebrow": "Приглашение на выставку",
    "title": "Выставка для клиентов компании",
    "description": "Короткое описание...",
    "date": "15 мая 2026",
    "location": "Москва, МВЦ «Экспоцентр»",
    "format": "Офлайн",
    "caption": "Встречайтесь с экспертами..."
  },
  "about": {
    "title": "Почему стоит прийти",
    "subtitle": "Выставка создана специально...",
    "whoText": "Кому это будет полезно",
    "valueList": ["Пункт 1", "Пункт 2"],
    "whyText": "Почему стоит зарегистрироваться..."
  },
  "heroImage": {
    "url": "./assets/hero-photo.png",
    "alt": "Фото с выставки"
  },
  "registration": {
    "subtitle": "Заполните форму...",
    "hint": "Нажимая кнопку, вы соглашаетесь...",
    "contact": { "email": "sales@company.ru", "phone": "+7 (495) 000-00-00" },
    "success": "Регистрация успешно отправлена...",
    "error": "Не удалось отправить заявку..."
  },
  "footer": { "year": 2026 }
}
```

> Важно: фронтенд ожидает наличие полей из разделов `event`, `about`, `registration`. Если backend отдаёт другие ключи — их нужно маппить на этот формат (см. `src/components/renderLanding.js`).

## 2) Отправка регистрации

`POST {apiBaseUrl}/registrations`

### Тело запроса (JSON)
```json
{
  "eventId": "expo-landing_V1",
  "inn": "7707083893",
  "company": "ООО «Пример»",
  "name": "Иванов Иван Иванович",
  "email": "name@company.ru"
}
```

### Ответ
Фронтенд пока ожидает `res.ok` и затем пытается прочитать JSON. Минимально:
```json
{ "ok": true }
```

Если ваш API возвращает другой формат — обновите `src/api/eventApi.js` (функцию `submitRegistration`).
