# Документация по лендингу Expo Landing

Лендинг сделан как статическая страница (GitHub Pages / Netlify static) и работает в браузере без сборщиков.
Данные для конкретного мероприятия подгружаются через API (в коде уже предусмотрено подключение), а при отсутствии backend используется mock-логика.

## Как устроен проект

- `index.html` — каркас страницы и семантическая разметка. Важно: все поля формы имеют фиксированные `id`, которые использует JS.
- `styles.css` — стили лендинга.
- `src/main.js` — точка входа: подгружает данные, отрисовывает их, и инициализирует форму.
- `src/config/landingConfig.js` — конфигурация конкретного лендинга (API и `eventId`).
- `src/api/eventApi.js` — клиент API: загрузка данных и отправка регистрации (пока с mock fallback).
- `src/api/mockEventApi.js` — мок для фронтенда без backend.
- `src/content/defaultEventData.js` — дефолтные данные мероприятия для отладки.
- `src/components/renderLanding.js` — маппинг данных мероприятия в DOM.
- `src/components/registrationForm.js` — логика формы: валидация, ошибки, уведомления и UX.
- `src/components/smoothScroll.js` — плавный скролл к форме.
- `src/lib/validators.js` — валидаторы (ИНН, email, обязательные поля).

## Подключение backend (под будущие мероприятия)

1. В `src/config/landingConfig.js` укажите:
   - `apiBaseUrl` — базовый URL вашего API
   - `eventId` — идентификатор мероприятия в БД
2. Реализуйте API согласно контракту (см. `docs/API_CONTRACT.md`).
3. После этого лендинг будет автоматически подставлять данные из БД и отправлять регистрацию на сервер.

## Как сделать “разные лендинги под разные мероприятия”

Есть два удобных сценария:

1. Разные деплои (несколько репозиториев/проектов)  
   Вы меняете `eventId` и `apiBaseUrl` в `src/config/landingConfig.js` под каждый лендинг.

2. Один деплой, разные мероприятия через query-параметр  
   Код поддерживает `?eventId=...` (например: `https://site/.../index.html?eventId=expo-123`).
   Тогда достаточно держать один сборник/деплой, а разные мероприятия выбирать параметром.

## Участие в форме регистрации

Форма имеет обязательные поля:
`ИНН`, `Название компании`, `ФИО представителя`, `Email`.

При ошибках — сообщения показываются под полями.
При успешной отправке — уведомление:
«Регистрация успешно отправлена. Пожалуйста, ожидайте письмо на указанную почту.»
